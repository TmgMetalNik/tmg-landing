# База знаний проекта TMG · Яндекс.Директ

Это рабочая память по проекту. Перед тем как писать новые объявления / тексты / БС / ТЗ креативов — **сначала читать эти файлы**.

## Файлы

- [`approved.md`](approved.md) — что согласовано с клиентом и можно использовать. Кубики, поинты, готовые шаблоны.
- [`rejected.md`](rejected.md) — что **НЕ зашло** клиенту (или мы выдумали). Сюда смотреть в первую очередь, чтобы не повторять ошибок.
- [`client-feedback-log.md`](client-feedback-log.md) — хронология решений по чатам с клиентом (Евгений Taikyuu).
- [`facts-to-confirm.md`](facts-to-confirm.md) — факты, требующие подтверждения (использовать осторожно или уточнять у клиента).

## Главные правила

1. **Не выдумывать цифры и факты.** Любая цифра (срок, процент, цена) — либо из `approved.md`, либо уточнять у клиента. Запрет на «3 дня», «1 день», «−30% к импорту», «+40% ресурса» если этого нет в approved.
2. **«СОЖ»** — обязательное слово в каждом заголовке Поиска (ключевое для семантики).
3. В тексте СОЖ можно опускать (есть в заголовке). Тексты должны **дополнять** заголовок, а не повторять.
4. Гео в текстах не упоминаем — пишем общефедерально, гео настраиваем как таргет в Директе.
5. Имена конкурентов (Лукойл, Blaser, Castrol, Mobil, Shell и др.) — **только в ключах**, в текстах НЕ упоминаем (модерация + юридика).

## Для кого эта база

- Агент `yandex-direct` (см. `../prompt.md`) — обязан читать перед запуском
- Любой исполнитель / копирайтер на проекте TMG
- Менеджер при согласовании с клиентом

## Когда обновлять

- После любого нового решения от клиента (Евгений) — добавлять в `client-feedback-log.md`
- Если поинт принят — переносить в `approved.md`
- Если поинт отклонён или мы выдумали — переносить в `rejected.md` с пометкой почему
- Факты, по которым нет подтверждения, но мы их используем — в `facts-to-confirm.md`
