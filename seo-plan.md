> Версия: 0.1 (draft). Создан 2026-05-24.
> Принцип работы: пункты идём поочерёдно, каждый раскрываем до конкретных задач с привязкой к артефактам. По мере прохождения помечаем статус.

## 0. Контекст

- **Домен:** theosmg.ru
- **Бизнес:** TMG · Theos Metal Group — производитель СОЖ в Санкт-Петербурге.
- **Продукты:** CutMax, AquaMax, AquaLite + комплексы Taikyuu-X (модификаторы поверхностей трения).
- **Стадия:** лендинг живой, реклама в Яндекс.Директ запущена, заявок мало.
- **Целевой результат SEO:** стабильный поток органических B2B-лидов (производства, технологи, снабжение).

**Цель:** убрать барьеры для индексации и ранжирования.

- [ ] Аудит индексации в Яндекс.Вебмастер: сколько страниц в индексе, есть ли ошибки сканирования
- [ ] Проверка `robots.txt` и `sitemap.xml` на корректность
- [ ] Скорость загрузки (LCP, CLS) — особое внимание hero-видео

- [ ] HTTPS, отсутствие смешанного контента, корректные 301/404
- [ ] Цели в Метрике: заявка, клик по телефону/email, скролл, время на странице, открытие каталога

**Артефакт:** `/root/tmg-landing/seo-audit-technical.md` (создаётся в ходе работы).

## 2. Сбор и кластеризация семантики ← **СЕЙЧАС ЗДЕСЬ**

**Цель:** собрать полное семантическое ядро и разложить по интентам, чтобы понять, под какие запросы строить страницы.

**Источники данных:**
- keyso — `get_domain_overview`, `get_organic_keywords_for_domain` (свой и конкуренты), `get_keyword_info`
- Яндекс.Вебмастер — `get_search_queries` (реальные показы/клики из Яндекса)
- Яндекс.Wordstat — частотности (через ручную выгрузку, если потребуется)

**Шаги:**
- [ ] Срез текущей видимости theosmg.ru (что Яндекс знает о сайте сейчас)
- [ ] Определение конкурентов через keyso (`get_domain_competitors`)
- [ ] Выгрузка органической семантики 5-7 ключевых конкурентов
- [ ] Дополнительные сидовые запросы по продуктам и применениям
- [ ] Кластеризация ядра по интенту:
  - **Коммерческие:** «купить СОЖ», «СОЖ цена», «производитель СОЖ», «поставщик СОЖ»
  - **Транзакционные узкие:** «полусинтетическая СОЖ», «эмульсия для шлифовки», «СОЖ для титана», «СОЖ для нержавейки»…
  - **Информационные:** «как выбрать СОЖ», «срок службы СОЖ», «утилизация СОЖ», «расчёт концентрации СОЖ»
  - **Брендовые:** свои (CutMax, AquaMax, AquaLite, Taikyuu) + чужие (конкуренты, импорт-аналоги)
- [ ] Приоритизация кластеров: частотность × конкуренция × близость к покупке

**Артефакт:** `/root/tmg-landing/seo-keywords.md` (ядро + кластеры) и `/root/tmg-landing/seo-competitors.md` (срез конкурентов).

## 3. Структура сайта (от лендинга к сайту-каталогу)

**Цель:** под каждый коммерческий/транзакционный кластер — отдельная посадочная страница.

- [ ] `/products/cutmax/`, `/products/aquamax/`, `/products/aqualite/` — карточки продуктов с ТТХ, применениями, ценой/запросом КП, даташитом PDF
- [ ] `/applications/` — посадки по типу операции: точение, фрезерование, шлифовка, нарезание резьбы, сверление…
- [ ] `/industries/` — по отраслям: машиностроение, авиапром, автопром, инструментальное производство
- [ ] `/materials/` — по обрабатываемому материалу: сталь, нержавейка, титан, алюминий, чугун…
- [ ] `/blog/` или `/baza-znaniy/` — статьи под информационные запросы
- [ ] `/cases/` — кейсы внедрений (реальные клиенты, цифры)
- [ ] `/contacts/`, `/about/`, `/dostavka/`, `/garantii/` — служебные

**Артефакт:** `/root/tmg-landing/seo-sitemap.md` (карта планируемых посадок с привязкой к кластерам).

## 4. Контент-стратегия

**Цель:** под информационные запросы и средне-частотные коммерческие — полноценный контент, который заодно даёт ссылки и поведенческие.

- [ ] Контент-план на 3-6 месяцев: 1-2 статьи в неделю
- [ ] Форматы:
  - Подборки «Как выбрать СОЖ для [операция / материал]»
  - Сравнения: своя СОЖ vs импорт-аналоги, эмульсия vs масляная СОЖ
  - Гайды: расчёт концентрации, контроль состояния, утилизация
  - Технические PDF-даташиты на каждый продукт (под скачивание = лид)
  - Калькуляторы: концентрация, расход, экономия — отлично ловят ссылки
- [ ] Видео в Rutube/YouTube + Дзен по тем же темам (доп. касание + ссылки)

**Артефакт:** `/root/tmg-landing/seo-content-plan.md`.
