# Инструкция по разработке сайта theosmg.ru

> Как устроен проект, где что живёт и как правильно деплоить. Читать перед любыми правками и заливкой файлов.

## Архитектура: где что живёт

| Где | Что | Назначение |
|---|---|---|
| **Хостинг** reg.ru `www/theosmg.ru/` | боевой сайт: html, css, js, php, `assets/`, PDF | то, что отдаётся посетителям. **Единственный источник правды для самого сайта.** |
| **Наш сервер** `/root/tmg-landing/` | рабочая копия-исходник + внутренние материалы + тулинг | мастерская: тут я редактирую и отсюда деплою |

**На нашем сервере держим только:**
- рабочую копию файлов сайта (то, что редактирую и заливаю);
- внутренние материалы, которым НЕЛЬЗЯ быть публичными (`ca-analysis/`, `*-plan.md`, `design-brief.md`, knowledge-base, токены, `.netrc`, `.mcp.json`);
- тулинг, который хостинг не потянет (`agents/yandex-direct/`, MCP, cron).

## Правила работы с файлами (ВАЖНО)

1. **Файл уже на хостинге → перемещаем ВНУТРИ хостинга, не гоняем через наш сервер.**
   Клиентские файлы часто заливают в `/_uploads/` (это вне docroot, по вебу недоступно). Чтобы файл стал доступен — `mv` на стороне хостинга в `www/theosmg.ru/...`, а НЕ «скачать к нам → залить обратно».
   ```bash
   lftp -u u3495982,<pass> -e "set ssl:verify-certificate no; set ftp:ssl-allow yes; set ftp:passive-mode yes; \
     mv '_uploads/Имя файла.pdf' 'www/theosmg.ru/assets/docs/imya-fajla.pdf'; bye" server273.hosting.reg.ru
   ```

2. **Ассеты сайта живут на хостинге.** Локально дублируем только то, что реально редактируем/деплоим. Публичные документы (паспорта, каталоги) после заливки на хостинг локально держать не обязательно.

3. **Внутренние материалы — никогда в docroot.** Анализ, планы, брифы, токены лежат у нас. Если положить их в `www/theosmg.ru/` — станут доступны по прямой ссылке.

4. **Имена файлов — URL-safe.** Без пробелов и кириллицы: `pasport-bezopasnosti.pdf`, а не `Pasport bezopasnosti.pdf`.

5. **Заменяешь версию ассета — удаляй старую из ОБОИХ мест** (и локально, и на хостинге). Иначе копятся мёртвые `*-v2`, `*-old`, дубли по 3 МБ.

6. **Фото — в WebP/JPEG, не PNG.** PNG только для логотипов/иконок/прозрачности. Тяжёлый PNG на первом экране бьёт по LCP и SEO.

## Деплой

- Доступ: FTPS, хост `server273.hosting.reg.ru`, логин `u3495982`, пароль в `/root/.netrc` (chmod 600). Порт 21 закрыт — только FTPS, пассивный режим.
- Корень сайта: `www/theosmg.ru/`.
- Залить файл:
  ```bash
  lftp -u u3495982,<pass> -e "set ssl:verify-certificate no; set ftp:ssl-allow yes; set ftp:passive-mode yes; \
    cd www/theosmg.ru/; put /root/tmg-landing/index.html; bye" server273.hosting.reg.ru
  ```
- После деплоя — проверять `curl -sI https://theosmg.ru/<путь>` (код ответа, content-type).
- Бэкап старой версии прода (до v2): `/root/tmg-landing/backup-prod-20260426/`.
- Резерв: Cloudflare Pages `https://tmgs-v4.pages.dev`.
